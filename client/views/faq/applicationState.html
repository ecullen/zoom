Status Meaning
The last stop/restart command returned a non-zero exit code.
The sentinel agent has lost connection to Zookeeper.
The 'notify' action has triggered. Most commonly this means that the application has crashed.
Two sentinel agents have configs with the same ID. The configs will need to be updated and the agents will need to be restarted to resolve.
Someone has indicated that this application can be ignored. It could be up or down.
Everything is working as expected.
The application is Read-Only. This means it is brought up/down outside of the normal Zookeeper startup. Zoom only reports whether it is up or down.
The sentinel mode has been set to 'react.' This means that it will react to dependency changes. You will most often see this if you send a dep_restart (restart with dependencies).
